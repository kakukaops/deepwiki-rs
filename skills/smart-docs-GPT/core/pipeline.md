# Documentation Pipeline

The documentation process is composed of five mandatory phases.

Phases MUST be executed in order.

## Phase List

1. Discovery  
2. Architecture Analysis  
3. Documentation Generation  
4. Diagram Synthesis  
5. Quality Review  

Language plugins MAY:
- Add heuristics
- Add focus areas
- Extend outputs

Language plugins MUST NOT:
- Reorder phases
- Remove mandatory outputs
- Break template contracts
