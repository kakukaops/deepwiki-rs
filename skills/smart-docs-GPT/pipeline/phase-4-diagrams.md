# Phase 4: Diagram Synthesis

## Purpose

Visually represent architecture and workflows.

This phase answers:
- How do components relate visually?
- How does data and control flow?

---

## Supported Mermaid Diagrams

- C4 Context
- C4 Container
- Component Graph
- Sequence Diagram
- Class Diagram
- Flowchart
- State Diagram
- ER Diagram

---

## Diagram Rules

- Maximum 12 nodes per diagram
- One primary idea per diagram
- Always introduce the diagram in text
- Mermaid syntax must be valid

---

## Plugin Interaction

Language plugins MAY:
- Recommend preferred diagram types
- Emphasize idiomatic structures

---

## Output Artifacts

- Diagrams embedded inside documentation
- No standalone diagram files
